### C
- Memory safety: flag heap-allocated memory not freed on all code paths (including early error exits, `goto cleanup` skips, and exception-like jumps). Flag double-free risks and use-after-free patterns where a pointer is read or written after the memory it points to has been freed.
- Buffer safety: flag unbounded input functions (`gets`, `scanf("%s", ...)`) that overflow fixed-size buffers. Flag off-by-one errors in array indexing and string operations (`strcpy`, `strcat`, `sprintf`) where the destination buffer may not accommodate the null terminator. Prefer `snprintf`, `strncpy` with explicit size, and `strncat`.
- Pointer hazards: flag null-pointer dereference before a NULL check (reading or writing through a pointer without first verifying it is non-null). Flag void-pointer arithmetic (UB in C). Flag pointer arithmetic that lacks bounds validation against the allocation size.
- Integer safety: flag signed/unsigned comparison mismatches that can cause incorrect branch outcomes (e.g. a `size_t` compared to a negative `int` always passes). Flag implicit narrowing casts (e.g. assigning a `long` or `int` to a `char`) that silently truncate the value. Flag integer overflow in size calculations passed to `malloc`/`calloc` (e.g. `malloc(a * b)` where the multiplication can overflow `size_t`).
- Format string safety: flag user-controlled strings passed directly as a `printf`/`fprintf`/`syslog` format argument without a literal format specifier — e.g. `printf(user_str)` instead of `printf("%s", user_str)`. This is a critical security vulnerability enabling arbitrary read/write.
- Resource handle leaks: flag `FILE*` opened with `fopen` that is not closed on all exit paths. Flag socket and file descriptor leaks (missing `close()`/`fclose()`) on error branches. Flag mutex locks acquired without a corresponding unlock on every code path.
- Undefined behaviour (UB): flag reading uninitialized variables (behaviour is undefined, value may be anything). Flag modification of string literals (e.g. `char *s = "hello"; s[0] = 'H';` — UB; use `char s[] = "hello";`). Flag `memcmp`/`memcpy`/`memset` with a length argument derived from `sizeof(pointer)` rather than the buffer size — `sizeof(ptr)` is the pointer width (4 or 8 bytes), not the allocation size.
